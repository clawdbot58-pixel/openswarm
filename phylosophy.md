# Philosophy

## 1. "It Just Works"

Software should work reliably. If it doesn't solve the problem effectively, it's not finished. "Good enough" is the enemy of great.

## 2. Make It Work, Then Polish

Move fast to get something functional. Visuals and polish come second—functionality first. You can't polish a broken foundation.

## 3. Clear Formats

Be explicit. Use consistent patterns. Ambiguity in structure leads to confusion in code and in team communication.

## 4. Build Piece by Piece

Compose functionality from small, focused modules. Each piece should do one thing well. Integration is where the magic happens.

## 5. Don't Rebuild the Wheel

Inspiration is fine—copying is okay if you translate it into your own context. We took inspiration from OpenClaw, but tried to reinvent too much and ended up with a half-baked nothing. Learn from what works, adapt it, make it yours.
